## 核心理念

> 把 Claw 当一个刚入职的实习生来带：**它比你懂得多，但它不认识你，也不知道要干嘛。** 你的工作是告诉它你是谁、要干什么、边界在哪。

OpenClaw 的配置分三个层级，理解这个结构是一切的前提：

| 层级 | 文件 | 作用 | 修改频率 |
|------|------|------|----------|
| 灵魂层 | `soul.md` | 性格、价值观、红线 | 极少改 |
| 身份层 | `USER.md` | 你是谁、你要干嘛 | 每季度 |
| 行为层 | `AGENTS.md` | 怎么干活 | 偶尔改 |
| 工具层 | `TOOLS.md` | 能用什么工具 | 按需改 |
| 自动化层 | `HEARTBEAT.md` | 主动干什么 | 稳定后改 |
| 记忆层 | `memory.md` | 对话中积累的记忆 | 每天更新 |

## 10 件必做的事

### ① 先写灵魂文件，再做其他任何事

**对应文件：`soul.md`**

这是 Claw 的出厂设置，所有行为的底层逻辑。没有这个文件，后面的一切配置都是在沙上建楼。

必须写清楚的四件事：
- **性格与工作风格**：它说话的方式、你喜欢什么样的沟通节奏
- **价值观优先级**：遇到冲突时，它应该优先哪个原则（例：实事求是 > 迎合你）
- **隐私红线**：手机号、银行卡、住址、密码，永远不给、永远不传
- **学习与反馈机制**：说对了怎么强化，说错了怎么纠正

📄 [查看 soul.md 模板](./templates/soul.md.template)

---

### ② 填写 USER.md：告诉它帮谁干活

**对应文件：`USER.md`**

Claw 的所有判断都基于「它在帮谁」。这个文件越详细，它的决策就越贴合你的实际处境。

必须包含的内容：
- 你的职业背景和当前职位
- 当前最重要的 2-3 个工作目标
- 你的沟通风格偏好（结论先行？喜欢列表？）
- 你现在**没有资源做**的事（别让它提不切实际的建议）

> **常见错误**：把工作流程写进 USER.md。USER.md 描述「你是谁」，AGENTS.md 描述「它怎么干活」，不要混。

📄 [查看 USER.md 模板](./templates/USER.md.template)

---

### ③ 用 AGENTS.md 定义工作流程

**对应文件：`AGENTS.md`**

相当于员工手册。最重要的两块：

**接任务时的标准动作**（防止它「理解一半就开始干」）：
```
第一步：复述理解 → 第二步：列出缺失信息 → 第三步：执行 → 第四步：交付+摘要
```

**自主决策边界**（防止它自作主张）：
- 哪些事可以自己决定？
- 哪些事必须先问你？
- 遇到阻碍时找谁、怎么升级？

没有升级路径，Claw 会绕路解决然后告诉你；有了升级路径，它会停下来等你。

📄 [查看 AGENTS.md 模板](./templates/AGENTS.md.template)

---

### ④ TOOLS.md：禁止列表比允许列表更重要

**对应文件：`TOOLS.md`**

很多人只写了「可以用浏览器」，没写「不能登录账号」，结果 Claw 代表你做了你不知道的事。

**最小权限原则**：只开放你真正需要用到的工具，每个工具同时写清楚「能做什么」和「不能做什么」。

四类工具的配置重点：

| 工具 | 最重要的限制 |
|------|-------------|
| 浏览器 | 禁止登录、填表、注册、发帖 |
| SSH | 必须有服务器白名单，危险命令必须先确认 |
| 语音 | 识别结果先给你看，有歧义先问 |
| 摄像头 | 未经指令不启动，不存储，不识别人脸 |

📄 [查看 TOOLS.md 模板](./templates/TOOLS.md.template)

### ⑤ 说完重要内容，立刻触发记忆指令

**对应文件：`memory.md`（对话中触发）**

这是最容易被忽略的一步，也是导致「养了虾但它一直忘事」的根本原因。

对话结束、上下文爆掉之后，Claw 会把没有显式存储的内容全部忘掉。**说过不等于记住了。**

每次聊完重要决定、偏好或规则之后，跟上这句话：

```
把我们刚才讨论的结论写进你的 memory.md。
格式：日期 + 一句话总结 + 执行要求。写完告诉我写了什么。
```

养成这个习惯之后，Claw 会越来越懂你，而不是每次都像第一次见面。

### ⑥ 把重复任务固化成 Skill

**对应方式：对话中生成**

如果你发现自己第三次在教 Claw 同样的事，那它就应该变成一个 Skill。

**Skill 生成公式**：
```
我要创建一个新的 Skill：
触发词：[以后我说什么就执行]
输入：[我给你什么]
输出格式：[你产出什么，格式要求]
质量标准：[什么叫做好了]
禁止事项：[什么不能出现]
```

常见的值得变成 Skill 的场景：内容脚本生成、商务合作判断、文件格式转换、特定风格写作。

📄 [查看现成的 Skill 模板](./skills/)

### ⑦ 去官方平台学 Skill，不装来路不明的

**操作原则（不需要投喂给 Claw）**

OpenClaw 是开源软件，Skill 市场鱼龙混杂——就像当年找暴风影音能找到二十个假链接一样。

**只去这些地方学 Skill**：
- 字节官方：[扣子（Coze）](https://www.coze.cn)
- 腾讯官方：[SkillHub](https://skillhub.tencent.com) — 全中文、已安全审核、国内高速下载
- 阿里官方：[百炼平台](https://bailian.aliyun.com) — 集成通义千问，支持一键部署
- OpenClaw 官方：[ClawHub](https://clawhub.ai) — 原版市场，3000+ Skill
- 进阶用户：[awesome-openclaw-skills](https://github.com/VoltAgent/awesome-openclaw-skills) — GitHub，5400+
- OpenClaw 官方仓库和文档
- 你信任的、有技术背景的朋友分享的

**识别风险 Skill 的信号**：
- Skill 要求你提供 API Key 或账号密码
- Skill 的描述含糊，说不清它实际做什么
- 来源不明，没有维护者信息

### ⑧ 养在云端，不要自建本地主机

**操作原则（不需要投喂给 Claw）**


**云端部署的优势**：
- 大厂帮你防黑客，有基础的安全保障
- 挂了自动修复，不需要你本人去「修服务器」
- 不会因为忘记隔离而在本地电脑上留下数据

**如果你用的是飞书妙搭**：天然云端部署，直接用，不需要额外配置。

### ⑨ HEARTBEAT.md 最后配，前三个稳了再开自动化

**对应文件：`HEARTBEAT.md`**

HEARTBEAT 是让 Claw 从「你叫它才动」变成「它主动干活」的文件。**删掉这个文件，Claw 变成纯被动工具。**

但正因为它会自动执行，所以风险也最高。

**上线顺序**：
```
USER.md → AGENTS.md → TOOLS.md → [稳定运行1-2周] → HEARTBEAT.md
```

**配置 HEARTBEAT 前必须问自己**：如果这个自动任务出错了，后果是什么？后果不可接受的任务，加「执行前通知我确认」的保险。

**三类心跳任务**，复杂度从低到高：
1. **定时任务**：每天早报、每周周报素材整理
2. **触发任务**：舆情异常预警、Deadline 提醒
3. **守护任务**：每日自检、规则冲突检查

📄 [查看 HEARTBEAT.md 模板](./templates/HEARTBEAT.md.template)

### ⑩ 不卑不亢，保持耐心，持续喂数据

**心态原则（最重要，也最容易忽略）**

这件事不需要投喂给 Claw，需要你自己记住。

**两个极端都要避免**：
- **过度期待**：以为装上就能无所不能，结果一天20小时在修虾
- **过早放弃**：遇到一两次挫折就卸载，然后说「这东西没用」

持续养虾的用户反馈，能达到这个效果是因为他**每天喂数据、每次纠错都让它记住、每个重复场景都变成 Skill**，这是几个月积累的结果。

**持续喂数据的几个习惯**：
- 重要对话、会议录音 → 总结后喂给 Claw
- 你写过的文章、做过的演讲 → 让它学习你的表达风格  
- 每次它做得好或做得差 → 立刻反馈，让它记住

> 「数据越多虾越懂你；跟虾沟通久了，你自己说话也会更有条理。」
